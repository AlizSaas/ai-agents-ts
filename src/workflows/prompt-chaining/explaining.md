# Prompt Chaining Workflow

## Overview
This folder demonstrates **prompt chaining with validation gates**, where multiple LLM calls are chained together sequentially, with each step's output feeding into the next, and quality gates in between to catch and recover from failures.

## Files

### index.ts
The main workflow implementing an AI-powered code generation pipeline with safety gates:

**Workflow Steps**:
1. **Read**: Load existing code from `bikes.ts`
2. **Generate**: LLM adds a new `getCheapest` function while preserving existing code
3. **Gate 1**: Verify the response actually contains the new function name
4. **Write**: Save the modified code to `bikes.ts`
5. **Gate 2**: Run `test.ts` to verify both old and new functions work correctly
6. **Rollback**: If tests fail, restore the original code
7. **Changelog**: LLM generates a changelog entry comparing old vs new code
8. **Append**: Save changelog entry to `changelog.txt`

**Key Features**:
- Automated rollback on test failure
- Multiple validation checkpoints
- Strips markdown code fences from LLM output
- Chain of LLM calls where each depends on the previous

### bikes.ts
The code file being modified by the workflow:
- Defines a `Bikes` type
- Contains `getTopExpensiveBikes` function (original)
- Contains `getCheapest` function (added by AI workflow)
- Both functions sort bikes by price and return top N items

### test.ts
Test file that validates the generated code:
- Tests `getTopExpensiveBikes` function
- Ensures correct sorting and slicing
- Acts as Gate 2 in the validation pipeline
- Simple assertion-based testing

### changelog.txt
Auto-generated changelog showing AI modifications:
- Records what was added in each workflow run
- Generated by comparing old vs new code
- Demonstrates audit trail for AI changes

## Pattern
This demonstrates the **Chain → Gate → Rollback** pattern:
1. Sequential LLM calls where output of one feeds the next
2. Validation gates between steps to catch bad AI output
3. Automated rollback mechanisms to recover from failures
4. Ideal for multi-step code modifications with safety requirements
